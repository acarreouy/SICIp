# SICIp
Sistema de Inteligencia Científica Institucional en Psicología: se trata de un sistema basado e un servicio Web para la investigación de la producción científico académica de la Facultad de Psicología y la Psicología nacional
En una primera etapa se trata de elaborar una herramienta que se comportara como un sistema completo de ingreso, procesamiento, investigación y elaboración de informes. El que permitirá encontrar rápidamente publicaciones, investigaciones, temas más relevantes, identificar expertos, acceder a datos y métricas confiables de los productores, podrá emitir informes bajo criterios preestablecidos. Su cometido es respaldar las estrategias de investigación, de los usuarios, dar visibilidad a la producción de la Facultad y emitir datos confiables para la toma de decisiones.

En una segunda etapa el sistema escalara hacia investigadores, investigaciones, publicaciones y métricas de la comunidad de expertos nacionales.

Debe combinar los diferentes formatos y productos de los, autores, afiliación, temas, relaciones y referencias, con datos enriquecidos y contenido vinculado. 
De este proceso se desprenderán informes cualitativos respaldados en datos métricos, y cuantitativos puros, acerca de la producción individual e institucional.

Se proyecta una base de datos que interaccione con una página web responsive y diferentes software para visualización de resultados, de modo de proporcionar en una visión rápida el estado en un aspecto determinado.

En síntesis esta página interactúa con una base de datos y formularios de ingreso que servirá para conocer la producción científica de los autores de la Facultad. 

Debe ser una página responsive, desarrollada en código abierto y escalable.
Debe soportar contenido dinámico (gráficas, animaciones, etc).
Debe recuperar toda la producción científico académica de la Facultad, creada y/o publicada en el país o en el exterior.
Debe servir para respaldar las estrategias de investigación de los usuarios.
Debe combinar datos enriquecidos, producción científico académica, autores, referencias y citas.
Debe vincular contenidos de producción, impacto, currículum, etc.
Debe producir métricas e informes de autores y/o temas.
Debe producir métricas e informes de producción y/o citas.
En la medida que tratamos que se constituya como un sistema debe permitir el ingreso de los datos en una forma eficiente y amigable.
Permite tener en un solo lugar toda la producción de la comunidad de autores de la Facultad.
Permite realizar búsquedas sobre la base de datos, aplicar filtros y obtener datos estructurados.
Permite evaluar la producción individual, grupal e institucional.
Permite identificar redes.
Permite análisis de palabras y métricas.
Permite la exportación de contenido métrico a planillas de calculo (se recomienda .csv)
Permite acceder a textos completos mediante enlaces a terceros.

* No permite la cosecha por conglomerados editoriales.
* No es un RI por tanto no contiene en si misma los textos completos, establecerá los enlaces a los mismos.

Tecnología (sujeta a evaluación):
Lenguajes HTML y PHP.
Hojas de estilo CSS.
Base de datos MYSQL.
Administración de la base PHPMYADMIN
Visualización de gráficas y redes HIGHCHARTS o PAJEK
Minería de datos PYTHON.

Proyecciones:
Creación de una App.
Creación de robots para búsqueda y recuperación (se plantea un escaneo semestral de la red).
Utilización de altmétricas.
Creación de un repositorio de datos de investigación.
Ver posibilidad de integrar resultados de Publish or Perish y Google Scholar.

Requerimientos técnicos:
Servidor físico o en la nube (APACHE) 1 TB.
Memoria RAM va a depender del tráfico (8 GB para inicio).
Base de datos MYSQL.
Administración de la base PHPMYADMIN

**Lo que van a ver es una maqueta que representa el estudio de los artículos de una revista cientifica y las diferentes mediciones y representaciones que se pueden realizar, es solo un acercamiento al tema sin terminar.
